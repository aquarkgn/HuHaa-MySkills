# HuHaa-MySkills 页面重构规划 v2.0

**日期**: 2026-06-29  
**原因**: 用户反馈设计存在严重的可用性问题  
**决策**: 完全重新设计整体布局和交互

---

## 🔴 当前设计的核心问题

### 1. **浮窗无法关闭**
- 问题：点击背景（modal-backdrop）没有正确关闭 Detail Modal
- 根本原因：Vue Teleport 结构问题或事件委托失效
- 影响：用户卡住，无法返回列表

### 2. **布局遮挡，用户体验极差**
- 问题：Detail Modal 占据 50vw（桌面），完全遮挡了 List 和 Sidebar
- 影响：看不到其他技能卡片，无法对比，工作流中断
- 用户期望：一个轻量级的 Side Panel 或 Drawer，不遮挡主内容

### 3. **菜单设计反人类**
- 问题：5 个独立下拉框（编辑器、类型、来源、产品、分类）
  - 每个下拉框占一行
  - 需要逐个打开、逐个选择
  - 选择后要再选下一个
  - 非常低效
- 用户期望：
  - 快速多选过滤（Chip 过滤器）
  - 搜索 + 过滤组合
  - Tag-based quick filter

### 4. **清理（Clear）按钮位置差**
- 问题：清理按钮在右侧顶部，与搜索框分离
- 影响：用户难以发现，违反 UI 习惯
- 用户期望：在搜索框旁边，或搜索框内有清除图标

### 5. **Logo 路径信息未显示**
- 问题：技能详情中未显示文件路径（虽然数据中有 `path` 字段）
- 影响：用户无法快速定位技能文件位置
- 用户期望：在详情中显示完整路径，可复制

## 🎯 新设计方案（v2.0）

### 整体布局

```
┌─────────────────────────────────────────────────────────┐
│  🔍 搜索框 [搜索关键字...]  [清空]  [扫描]  [导出]      │  (搜索和快速操作在顶部)
├─────────────────────────────────────────────────────────┤
│ ✓ Hermes ✓ Claude Code ✓ Codex   [更多过滤]             │  (Chips 快速多选)
├─────────────────────────────────────────────────────────┤
│                                    │                     │
│  📋 技能列表 (Grid)               │  技能详情 Panel      │
│  ┌─────────────────────────┐     │  (侧面板，不遮挡)    │
│  │ 卡片 1 │ 卡片 2 │ 卡片 3│     │                     │
│  ├─────────────────────────┤     │  名称 / 描述         │
│  │ 卡片 4 │ 卡片 5 │ 卡片 6│     │  📁 路径显示         │
│  │                         │     │  [复制] [打开]       │
│  └─────────────────────────┘     │                     │
│                                    │                     │
└────────────────────────────────────┴─────────────────────┘
```

### 核心改进

#### 1. **搜索和操作栏**（新）
```
[🔍 搜索框 placeholder="名称、描述、路径..."]  [清空]  [扫描]  [统计]
```
- 搜索框内包含清空按钮 (X icon)
- 快速操作按钮：扫描、统计、导出等
- 单行设计，节省空间

#### 2. **Chip 过滤器**（替代下拉框）
```
✓ Hermes (151)  ✓ Claude Code (60)  ✓ Codex (1)  [+3 more]  [清空过滤]
```
- 所有来源/编辑器显示为 Chips
- 点击 Chip 快速切换
- 支持多选
- [+more] 按钮展开更多选项
- 清除所有过滤只需一次点击

#### 3. **列表区域**（改进）
- Grid 卡片布局（保留）
- 响应式列数（保留）
- **BUT**: 右侧始终显示一个侧面板（Detail Drawer）

#### 4. **Detail Drawer**（替代 Modal）
- 位置：右侧固定（不是浮窗）
- 宽度：25-35% 的窗口（或固定 300-400px）
- 特点：
  - 不遮挡列表，用户可继续浏览
  - 点击列表中的其他卡片立即切换
  - 关闭按钮清晰可见
  - 可滚动，显示完整内容

#### 5. **Detail Panel 内容**（改进）
```
┌─ Detail Drawer ──────────────┐
│  技能名称              [X] 关闭 │
│                               │
│  📚 描述                      │
│  Lorem ipsum...              │
│                               │
│  📁 路径信息 (新)             │
│  /Users/mac/.hermes/...  [📋] │
│                               │
│  🏷️ 标签                      │
│  skill | hermes | Hermes...  │
│                               │
│  ℹ️ 来源                      │
│  Hermes Skills Library (4)   │
│                               │
│  [打开文件] [复制路径] [编辑]  │
│                               │
└─────────────────────────────┘
```

## 📐 具体实现

### 文件改动

#### `packages/web/src/App.vue`
- 移除 Teleport Modal 实现
- 改为侧面板（Drawer）实现
  - 使用 CSS Grid 重新规划布局
  - 右侧固定 Drawer 区域
  - 列表和 Drawer 在同一层（不用浮窗）

#### `packages/web/src/styles.css`
- 重设 `.shell` 布局：`grid-template-columns: 1fr 300px`（或响应式）
- 重设 `.topbar` 设计
- 新增 `.filter-chips` 样式
- 重设 `.detail-drawer` 样式（侧面板）
- 移除所有 `.modal-backdrop` 样式
- 更新响应式断点

#### `packages/web/src/store.ts`（如果需要）
- 确保 `selected` 技能状态正确管理
