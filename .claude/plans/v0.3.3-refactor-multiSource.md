## 🎯 核心问题（诊断完成）

### 问题 1: 扫描逻辑严重退化
- **现象**: `npm run scan` 只返回 Hermes 技能（~160 条），其他 8 个源返回 0 文件
- **根本原因**: 配置中的源路径大多为空（`~/.claude/skills`, `~/.cursor`, etc.）或目录不存在
- **影响**: 用户看不到 Claude Code、Cursor、Codex、MCP、Hermes Plugin 等源的技能
- **关键数据**:
  ```
  hermes: 142 files ✅
  claude-code: 0 files (目录不存在或为空)
  cursor: 0 files
  codex: 0 files  
  mcp-config: 3 files (有数据但未展示)
  project-runbook: 11 files (有数据)
  hermes-plugin: 0 files
  skills: ? (未检验)
  mcp: ? (未检验)
  ```

### 问题 2: 过滤器混淆 editor/source
- **现象**: 前端过滤 dropdown 标签混乱；多源场景下统计错误
- **根本原因**: store 中混淆了 `byEditor` 和 `bySource` 的概念
  ```javascript
  // 行 37，skills.js
  editors(state) {
    return Object.entries(state.stats?.byEditor || state.stats?.bySource || {})
    // ❌ 这里用 || 是歧义的，应该明确只用 byEditor
  }
  ```
- **影响**: 过滤界面显示不清、用户不知道选哪个

### 问题 3: 页面布局不适配多源
- **现象**: 左侧筛选栏在多源场景下变得杂乱；badge/brand 显示无序
- **根本原因**: 
  1. 布局只针对 Hermes 单源优化
  2. `brand` 字段在 Hermes 中大多为空或不填
  3. 多源时 product/category 字段含义混乱

### 问题 4: Logo/品牌展示验证不合理
- **现象**: 页面右侧详情没有清晰的 source/editor/brand 视觉区分
- **根本原因**: 没有为每个源/编辑器设计对应的图标和配色
- **影响**: 用户无法快速识别技能来源

### 阶段 2: 过滤器和统计重构（E2）

**目标**: 清晰区分 editor / source / kind，实现正确的多维过滤

#### E2-1: 修复 store 统计逻辑
文件: `packages/web/src/stores/skills.js`

**问题代码** (第 37 行):
```javascript
editors(state) {
  return Object.entries(state.stats?.byEditor || state.stats?.bySource || {})
  // ❌ 歧义，应该明确
}
```

**修复方案**:
```javascript
// 1. 在服务端明确分离统计
// /api/stats 返回:
{
  bySource: { hermes: 142, 'claude-code': 10, cursor: 5, ... },
  byEditor: { 'Hermes Agent': 142, 'Claude Code': 10, 'Cursor': 5, ... },
  byKind: { skill: 150, instruction: 7, ... },
  byProduct: { ... },
  byBrand: { ... }
}

// 2. 在前端明确使用
editors(state) {
  // 优先用 byEditor，如果不存在用 bySource
  return Object.entries(state.stats?.byEditor || {})
    .map(([name, count]) => ({ name, count }));
}
```

**验收标准**:
- [ ] `editors` getter 返回正确的统计（不重复，不混淆）
- [ ] `filterSources` getter 与 `editors` 分离且独立
- [ ] 过滤时 editor 和 source 互不影响

#### E2-2: 修复过滤逻辑
文件: `packages/web/src/stores/skills.js` 第 60-76 行

**问题代码**:
```javascript
filtered(state) {
  let arr = state.skills;
  const { editor, kind, source, product, brand } = state.filters;
  if (editor) arr = arr.filter(x => (x.editor || x.source) === editor); // ❌ 混淆
  if (kind) arr = arr.filter(x => x.kind === kind);
  if (source) arr = arr.filter(x => x.source === source);
  // ...
}
```

**修复方案**:
```javascript
filtered(state) {
  let arr = state.skills;
  const { editor, kind, source, product, brand } = state.filters;
  
  // 分离过滤逻辑
  if (editor) {
    // editor 可能来自 x.editor 或 x.source（向后兼容）
    arr = arr.filter(x => {
      const itemEditor = x.editor || x.source;
      return itemEditor === editor;
    });
  }
  if (kind) arr = arr.filter(x => x.kind === kind);
  if (source) arr = arr.filter(x => x.source === source);
  if (product) arr = arr.filter(x => x.product === product);
  if (brand) arr = arr.filter(x => x.brand === brand);
  
  // ... (目录路径和全文搜索保持不变)
}
```

**验收标准**:
- [ ] 按 source 过滤能隔离出单个源的技能
- [ ] 按 editor 过滤能隔离出单个编辑器的技能
- [ ] 按 kind/product/brand 过滤正确
- [ ] 多个过滤条件组合时逻辑正确（AND，非 OR）
