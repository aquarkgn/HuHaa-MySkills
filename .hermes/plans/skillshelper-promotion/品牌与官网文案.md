## 官网信息架构与定稿文案

### 首屏

**眉标**

> 呼哈哈-技能助手 · 本地 AI 工作流能力中枢

**标题**

> AI 技能、插件、MCP 配置散落各处？
>
> 把它们变成你的能力工作台。

**说明**

> 扫描本机技能、插件、MCP 配置与项目运行手册，统一查看、搜索、分类、筛选；需要时查看中文对照，命令与代码保持原样。

**主 CTA**：`GitHub 仓库`

**次 CTA**：`快速开始`

**安装命令**

```bash
npm install -g skillshelper@latest
```

### 痛点区

**标题**

> AI 工作流失控，往往从“散”开始

**卡片一：能力散落在不同工具里，找不到也管不住**

> 技能目录、插件、MCP 配置和项目说明各有入口。装了什么、来自哪里、是否重复，全靠记忆与翻目录。

**卡片二：知道文件存在，却看不懂也无法判断该不该用**

> 说明常是英文且分散在原始文件中。整页机翻又会误伤命令和代码，结果是信息很多、决策很慢。

### 功能区

**标题**

> 从发现到维护，一处完成

**功能一：扫描多个本地来源，先看清你的能力版图**

> 发现编辑器技能、个人技能库、插件、MCP 配置与项目运行手册；扫描结果经过去重后进入同一份本地索引。

**功能二：按层级、来源和品牌整理，一秒定位**

> 以编辑器工具技能、个人技能库、其它来源构成三级结构，配合搜索、排序、分组和筛选，让查找不再依赖路径记忆。

**功能三：保留原文语境，需要时用中文理解**

> 支持中英对照与本地缓存；中文内容不会重复翻译，代码块与命令保留原样。MCP 配置在展示前会脱敏。

### 信任区

**标题**

> 为了可控，而不是为了更复杂

**说明**

> 文件扫描保持只读，MCP 配置展示前脱敏；每一项价值都能在本地运行中复核。

**事实卡**

- 多来源：技能、插件与配置统一索引
- 三级分类：来源结构一眼可见
- 本地缓存：翻译和图标避免重复处理
- 已发布 npm · MIT

### 快速开始区

**标题**

> 从本机开始，几分钟看见全局

**说明**

> 一行安装，一行启动。扫描与索引保留在本机；使用中文对照时才会请求翻译服务，结果会缓存到本地。

```bash
npm install -g skillshelper@latest
skillshelper start
```

## 禁用表达

- 不说“所有数据永不出本机”：用户触发翻译时会调用翻译服务。
- 不把“本机扫描到的数量”写成产品恒定能力。
- 不把未实时验证的版本、release 数、Star/Fork 数放入官网和封面。
- 不称为“聊天助手”或“万能 AI 平台”；聚焦能力发现、整理和理解。
